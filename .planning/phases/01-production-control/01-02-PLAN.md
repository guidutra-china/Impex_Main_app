  <behavior>
    - Test: getShipmentReadyQuantityByItem returns correct sums grouped by PI item (multiple items, multiple dates)
    - Test: entries with null actual_quantity are treated as 0 in sums but NOT as "delayed" in status
    - Test: entry with actual_quantity >= quantity is "on track or ahead"
    - Test: entry with actual_quantity > 0 but < quantity is "partial"
    - Test: entry with actual_quantity = 0 is "reported zero"
  </behavior>
  <action>
    1. Write unit tests in `tests/Unit/ShipmentReadyQuantityTest.php` covering the behavior above. Use `Model::create()` pattern, `RefreshDatabase` trait.

    2. Extend the admin `EntriesRelationManager.php` table — add these columns AFTER the existing `quantity` column:

       ```php
       TextColumn::make('actual_quantity')
           ->label(__('forms.labels.actual_quantity'))
           ->numeric()
           ->alignEnd()
           ->placeholder('—')
           ->badge()
           ->color(fn ($state, $record) => match (true) {
               $state === null => 'gray',                  // not reported
               $state >= $record->quantity => 'success',   // on or ahead
               $state > 0 => 'warning',                    // partial
               default => 'danger',                        // reported zero
           }),
       ```

    3. Add a `delta` virtual column after actual_quantity:
       ```php
       TextColumn::make('delta')
           ->label('Delta')
           ->getStateUsing(fn ($record) => $record->actual_quantity !== null
               ? $record->actual_quantity - $record->quantity
               : null)
           ->numeric()
           ->alignEnd()
           ->placeholder('—')
           ->color(fn ($state) => match (true) {
               $state === null => 'gray',
               $state >= 0 => 'success',
               default => 'danger',
           }),
       ```

    4. Also add `actual_quantity` to the admin form (so admin can also input actuals if needed):
       ```php
       TextInput::make('actual_quantity')
           ->label(__('forms.labels.actual_quantity'))
           ->numeric()
           ->minValue(0)
           ->nullable(),
       ```

    IMPORTANT: Do NOT change the admin form's existing fields (proforma_invoice_item_id, production_date, quantity) — only ADD the actual_quantity field. The admin has full edit access (unlike the supplier portal which only exposes actual_quantity).

    Run tests: `php artisan test --filter=ShipmentReadyQuantity`
  </action>
  <verify>
    <automated>php artisan test --filter=ShipmentReadyQuantity</automated>
  </verify>
