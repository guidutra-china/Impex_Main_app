# Phase 4 — UI Design Contract

> Visual and interaction contract for Phase 4: Redesign Production Schedule UX.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — Filament 4 semantic token system |
| Preset | not applicable |
| Component library | Filament 4 (filament/filament 4.7.2) — Schemas, Tables, Actions, Infolists |
| Icon library | Heroicons (bundled with Filament 4, `heroicon-o-*` and `heroicon-s-*`) |
| Font | Instrument Sans (declared in `resources/css/app.css` `@theme` block) |

**Notes:**
- This phase uses Laravel 12 / Filament 4 / Livewire 3, not React/Next.js/Vite. shadcn is out of scope.
- Styling is Tailwind CSS v4 (CSS-first config via `@theme` block) with Filament's semantic color tokens.
- Source: `resources/css/app.css` (font), `app/Providers/Filament/SupplierPortalPanelProvider.php` (primary color), RESEARCH.md (stack).

## Spacing Scale

Declared values (multiples of 4, matching Tailwind v4 defaults used in existing Blade partials):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding (`px-1`), inline status dots |
| sm | 8px | Compact table cell padding (`py-2`), button icon gap |
| md | 16px | Table cell horizontal padding (`px-4`), card body padding |
| lg | 24px | Section vertical gap (`space-y-6`), infolist section padding |
| xl | 32px | Between infolist sections, calendar strip outer padding |
| 2xl | 48px | Major section breaks on view pages |
| 3xl | 64px | Page-level top padding (Filament panel default) |

Exceptions:
- **Legacy — do not replicate:** Table row padding `py-2.5` (10px) and progress bar height `h-2.5` (10px) are confirmed in the existing partials `pi-production-progress.blade.php` and `shipment-ready-summary.blade.php`. These break the 4-point grid and must not be used in any new component built in this phase. Existing partials retain these values only to avoid visual regression.
- Touch targets for "Confirm Today" modal inputs: minimum 48px height (`h-12` Tailwind class) — satisfies WCAG 2.5.5 (44px minimum) and is a valid 4-point multiple.

## Typography

All sizes use Instrument Sans. Filament 4 applies this font via the app CSS `@theme` block; no per-component override needed.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 | Table cell content, helper text, relation manager rows |
| Label | 12px (`text-xs`) | 600 (semibold) | 1.4 | Table column headers, status badge labels, timeline date labels |
| Heading | 16px (`text-base`) | 600 (semibold) | 1.3 | Section headings in infolists, card titles, modal headings |
| Display | 20px (`text-xl`) | 600 (semibold) | 1.2 | Page title (Filament `ViewRecord` heading), widget stat values |

**Declared weights:** 400 (regular) and 600 (semibold). Do not introduce 500 (font-medium) or 700 in any new component. The Label role uses `font-semibold` (600), not `font-medium` (500). If an existing partial already uses `font-medium` for label-style text (confirmed in `pi-production-progress.blade.php`), leave it as-is to avoid visual regression — but do not carry that pattern forward.

**Numeric data:** All quantities and dates in tables use `tabular-nums` — confirmed existing pattern from `shipment-ready-summary.blade.php`.

## Color

The project has two panels with distinct primary colors. This phase touches both.

| Panel | Primary | Source |
|-------|---------|--------|
| Supplier Portal | Amber (`Color::Amber`) | `SupplierPortalPanelProvider.php` |
| Admin Panel | Indigo (`Color::Indigo`) | `AdminPanelProvider.php` |

Use Filament semantic tokens throughout — never hardcode hex values in Blade partials.

| Role | Filament Token | Usage |
|------|---------------|-------|
| Dominant (60%) | `bg-white dark:bg-gray-900` | Page background, table body rows |
| Secondary (30%) | `bg-gray-50 dark:bg-white/5` | Table headers, card surfaces, infolist section backgrounds, `details` summary rows |
| Accent (10%) | `text-primary-600 dark:text-primary-400` (Amber in portal, Indigo in admin) | "Confirm Today" button, "Submit for Approval" button, active progress bar fill, primary CTA buttons |
| Destructive | `text-danger-600 dark:text-danger-400` / `bg-danger-600` | Reject action button, past-due ETA flags in ComponentRiskWidget, zero-production cells |
| Success | `text-success-600 dark:text-success-400` | Approved status badge, on-target production cells, "Approve" action button |
| Warning | `text-warning-600 dark:text-warning-400` | Submitted status badge, partial-production cells (actual > 0 but below planned) |
| Neutral | `text-gray-400 dark:text-gray-500` | Not-yet-reported cells (null `actual_quantity`), empty state copy |

**Accent reserved for:** "Confirm Today" action button, "Submit for Approval" action button, filled segment of the progress bar. Do not use accent for status badges (use success/warning/danger/gray per state) or for table row hover states (use Filament's default hover tokens).

**Status badge color map** (from `ProductionScheduleStatus` enum in RESEARCH.md):

| Status | Color Token |
|--------|-------------|
| draft | gray |
| submitted | warning |
| approved | success |
| rejected | danger |

**Component status color map** (for `ComponentStatus` enum):

| Status | Color Token |
|--------|-------------|
| at_supplier | gray |
| at_factory | warning |
| in_transit | success |

## Component Inventory

All components use existing Filament 4 patterns — no new libraries introduced.

| Component | Filament Type | Location | Notes |
|-----------|--------------|----------|-------|
| Calendar/timeline strip | `ViewEntry` Blade partial | `resources/views/filament/production-schedule/calendar-timeline.blade.php` | Horizontal scrollable div, CSS grid, dates as columns, products as rows. Same pattern as `pi-production-progress.blade.php` |
| Confirm Today summary card | `ViewEntry` Blade partial | `resources/views/filament/production-schedule/confirm-today-summary.blade.php` | Shows today's planned entries; drives modal form |
| Confirm Today action | `Action` with `->form([...])` modal | Portal `ViewProductionSchedule::getHeaderActions()` | Visible only when today's entries exist |
| Submit for Approval action | `Action` with `->requiresConfirmation()` | Portal `ViewProductionSchedule::getHeaderActions()` | Visible only when `status === draft` |
| Approve/Reject actions | `Action` modal | Admin `ViewProductionSchedule::getHeaderActions()` | Visible only when `status === submitted` |
| Status badge | `TextColumn::make('status')->badge()` | List pages, infolist | Uses enum `color()` method |
| ComponentsRelationManager (portal) | `RelationManager` | Portal `ViewProductionSchedule` | Create/edit; no delete |
| ComponentsRelationManager (admin) | `RelationManager` | Admin `ViewProductionSchedule` | Read-only + risk flag column |
| Component risk widget | `StatsOverviewWidget` or custom `Page` section | Admin `ViewProductionSchedule` | N components late, earliest overdue ETA |

## Interaction States

### Calendar/Timeline Strip

| Cell State | Visual Treatment |
|-----------|-----------------|
| No entry for date | Empty cell, gray border |
| Entry — not yet reported (null actual) | `text-gray-400 dark:text-gray-500`, dash character "—" |
| Entry — zero actual reported | `text-danger-600 dark:text-danger-400`, bold "0" |
| Entry — partial (actual > 0, < planned) | `text-warning-600 dark:text-warning-400`, shows actual value |
| Entry — on target (actual >= planned) | `text-success-600 dark:text-success-400`, shows actual value |
| Today's column | Highlighted column header with `bg-primary-50 dark:bg-primary-900/20` background |

### Approval Workflow States

| Schedule Status | Supplier Portal View | Admin Panel View |
|----------------|---------------------|-----------------|
| draft | "Submit for Approval" button visible (primary/accent color) | No approval actions visible |
| submitted | Status badge only; no further supplier actions | "Approve" (success) and "Reject" (danger) buttons visible |
| approved | Status badge + `approved_at` date displayed | Status badge + approver name displayed |
| rejected | Status badge + `approval_notes` displayed | Status badge + `approval_notes` displayed |

### Component ETA Risk Flag

| Condition | Visual Treatment |
|-----------|-----------------|
| ETA is in the future | Default row styling |
| ETA is today | `text-warning-600 dark:text-warning-400`, "Due today" badge |
| ETA is past | `text-danger-600 dark:text-danger-400`, "Overdue" badge, row background `bg-danger-50 dark:bg-danger-900/10` |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (supplier — daily) | "Confirm Today's Production" |
| Primary CTA (supplier — submit) | "Submit for Approval" |
| Primary CTA (admin — approve) | "Approve Schedule" |
| Primary CTA (admin — reject) | "Reject Schedule" |
| Empty state — no entries for today | "No production planned for today. Check the calendar below for your next scheduled entry." |
| Empty state — no components | "No components added yet. Add components to track parts and materials for this schedule." |
| Empty state — no schedules (list page) | "No production schedules assigned. Contact your account manager if you expected to see a schedule here." |
| Error state — submit fails | "Could not submit this schedule. Only draft schedules can be submitted. Refresh the page and try again." |
| Error state — approve fails | "Could not approve this schedule. It must be in 'Submitted' status. Refresh to see the latest state." |
| Error state — component save fails | "Failed to save component. Check that all required fields are filled and try again." |
| Confirmation — submit for approval | "Submit this production schedule for admin review? You will not be able to edit entries after submitting." |
| Confirmation — approve | "Approve this production schedule? This will mark it as confirmed and notify the supplier." |
| Destructive confirmation — reject | Modal with textarea: "Reason for rejection (required)" — submit label: "Reject Schedule" |

**Destructive actions in this phase:**

| Action | Approach |
|--------|----------|
| Reject schedule | Modal action with required `approval_notes` textarea before confirming — `->form([Textarea::make('approval_notes')->label('Reason for rejection')->required()])` |
| No delete actions | Supplier cannot delete entries or components; admin cannot delete from the redesigned views |

## Layout Specification

### Supplier Portal — ViewProductionSchedule

```
[Header]
  Page title: "Production Schedule #NNN"
  Status badge (draft/submitted/approved/rejected)
  [Header Actions]
    - "Confirm Today's Production" (accent, visible when today's entries exist AND status === draft)
    - "Submit for Approval" (primary, visible when status === draft)

[Infolist Section: Schedule Details]
  - PI reference, supplier, date range, version

[Infolist Section: Production Calendar]
  - calendar-timeline.blade.php (ViewEntry, full width)
    Horizontal scrollable grid: rows = products, columns = dates

[Infolist Section: Today's Confirmation]
  - confirm-today-summary.blade.php (ViewEntry, full width)
    Visible only when today has planned entries

[Relation Manager Tab: Entries]
  - Table with date filter context
  - EditAction visible only when status === draft AND user has permission

[Relation Manager Tab: Components]
  - ComponentsRelationManager (portal)
  - CreateAction + EditAction visible when status === draft
```

### Admin Panel — ViewProductionSchedule

```
[Header]
  Page title: "Production Schedule #NNN — [Supplier Name]"
  Status badge
  [Header Actions]
    - "Approve Schedule" (success, visible when status === submitted)
    - "Reject Schedule" (danger, visible when status === submitted)

[Infolist Section: Schedule Details]
  - PI reference, supplier, submitted_at, approved_by, approved_at, approval_notes

[Infolist Section: Production Calendar]
  - calendar-timeline.blade.php (ViewEntry, full width)

[Widget: Component Risk Summary]
  - StatsOverviewWidget: N components overdue, earliest overdue ETA

[Relation Manager Tab: Entries]
  - Read-only table

[Relation Manager Tab: Components]
  - ComponentsRelationManager (admin) — read-only + ETA risk column
```
